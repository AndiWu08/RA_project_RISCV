Lessons learned / Erfahrungsbericht
- Code in Files unterteilen hilft der Übersichtlichkeit (nächstes Mal wird's besser)
- neue Funktionen in kleinen Teilen direkt testen, sonst wird die Fehlersuche nur komplexer
- Probleme dann lösen, wenn man sie lösen muss: 
    Beim durchlesen der Aufgabenstellung dachte ich jumps werden am kompliziertesten. 
    Nachdem die Code Base stand aber deutlich unkomplizierter als gedacht.
